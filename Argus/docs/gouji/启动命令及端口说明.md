# Argus Compound — 启动命令及端口说明

## 基础设施 (Docker)

```bash
# 启动 PostgreSQL + Redis + ChromaDB
docker-compose up -d
```

| 服务 | 端口 | 说明 |
|------|------|------|
| PostgreSQL (pgvector) | `5432` | 数据库 |
| Redis | `6379` | 缓存 |
| ChromaDB | `8000` | 向量存储 |

---

## 后端 — go-sensory (Go)

### 标准 HTTP 模式

```bash
cd go-sensory
go run ./cmd/server/main.go
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--port` | `8090` | API 服务端口 |
| `--fps` | `1` | 屏幕捕获帧率 (1-30) |
| `--backend` | `sck` | 捕获后端: `sck` / `cg` |
| `--shm` | `true` | 共享内存 IPC |
| `--open-browser` | `true` | 自动打开 Dashboard |
| `--vlm-config` | `""` | VLM 配置文件路径 |
| `--mcp` | `false` | MCP stdio 模式 |

**端口 `8090` 端点：**

| 端点 | 地址 |
|------|------|
| Dashboard | `http://localhost:8090` |
| WebSocket Frames | `ws://localhost:8090/ws/frames` |
| WebSocket Control | `ws://localhost:8090/ws/control` |
| Action REST API | `http://localhost:8090/api/action` |
| Status | `http://localhost:8090/api/status` |
| VLM Chat | `http://localhost:8090/v1/chat/completions` |
| VLM Health | `http://localhost:8090/api/vlm/health` |
| Config API | `http://localhost:8090/api/config/providers` |
| Pipeline Stats | `http://localhost:8090/api/pipeline/stats` |

### MCP stdio 模式

```bash
cd go-sensory
go run ./cmd/server/main.go --mcp
```

不使用网络端口，通过 stdin/stdout 进行 JSON-RPC 2.0 通信，日志输出到 stderr。

## 前端 — web-console (Next.js)

```bash
cd web-console
npm install        # 首次安装依赖
npm run dev        # 开发模式
```

| 命令 | 端口 | 说明 |
|------|------|------|
| `npm run dev` | `3090` | 开发模式 (热重载) |
| `npm run build` | — | 生产构建 |
| `npm run start` | `3090` | 生产模式 |

前端访问地址: `http://localhost:3090`

---

## 端口汇总

| 端口 | 服务 | 协议 |
|------|------|------|
| `3090` | web-console (Next.js) | HTTP |
| `5432` | PostgreSQL | TCP |
| `6379` | Redis | TCP |
| `8000` | ChromaDB | HTTP |
| `8090` | go-sensory (API + Dashboard) | HTTP/WS |
