# /audit — 模块审计工作流

// turbo-all

## 步骤 0：加载技能与规范

1. 读取技能入口：`.agent/skills/acosmi-refactor/SKILL.md`
2. 读取编码规范：`.agent/skills/acosmi-refactor/references/coding-standards.md`
3. 读取文档模板：`.agent/skills/acosmi-refactor/references/doc-template.md`（审计报告模板）

## 步骤 1：确定审计范围

1. 向用户确认审计目标（模块名、文件范围）
2. 列出目标模块的核心文件清单
3. 确认审计维度：
   - 代码质量（行数、函数长度、命名）
   - 错误处理（是否遗漏）
   - 安全性（FFI 内存管理、输入校验）
   - 架构合规（是否符合改造方案）

## 步骤 2：逐文件审查

1. 对每个核心文件进行审查
2. 按以下标准分级记录发现：
   - **P0（阻断）**：编译错误、严重安全漏洞
   - **P1（必修）**：逻辑 Bug、内存泄漏风险、功能缺失
   - **P2（建议）**：代码风格、可维护性改进

## 步骤 3：生成审计报告

1. 按审计报告模板生成文档：`docs/renwu/audit-YYYYMMDD-<module>.md`
2. 包含审计摘要、发现列表、延迟项
3. 对可立即修复的 P0/P1 项，提出具体修复方案

## 步骤 4：生成修复跟踪与 bootstrap

1. 创建任务跟踪文件：`docs/renwu/task-YYYYMMDD-<topic>.md`
   - 使用状态标记表格：⬜ 待做 / 🔄 进行中 / ✅ 完成 / ⏭️ 跳过
   - 列出所有待修复项，关联审计发现编号
   - 包含"完成后需更新的文档"清单
   - 包含变更记录
2. 创建 bootstrap 上下文：`docs/renwu/bootstrap-<topic>.md`
   - 引用任务跟踪文件（而非重复清单）
   - 包含关键文件索引
   - 包含新窗口启动指令

## 步骤 5：处理发现

1. 询问用户是否立即修复 P0/P1 项
2. 如需修复，切换到 `/refactor` 工作流执行
3. 将无法立即处理的项登记到 `docs/renwu/deferred-items.md`

## 步骤 6：更新文档

1. 确认审计报告已保存
2. 确认任务跟踪文件已创建
3. 更新延迟待办清单
4. 向用户汇报审计结果摘要，提供新窗口启动指令
