# 多平台内容发布策略

本技能为 oa-media 子智能体提供跨平台内容适配和发布策略的完整指南。

## 平台特性对比

| 维度 | 微信公众号 | 小红书 | 自有网站 |
|------|-----------|--------|---------|
| 内容形式 | 长图文 | 短图文/视频 | 长文/Blog |
| 标题限制 | ≤64 字符 | ≤20 字符 | 无限制 |
| 正文限制 | ≤50000 字 | ≤1000 字符 | 无限制 |
| 格式 | HTML | 纯文本 + emoji | Markdown |
| 图片限制 | ≤500 张 | ≤9 张 | 无限制 |
| 调性 | 专业/权威 | 真实/亲和 | 自定义 |
| 发布方式 | API 异步 | RPA | REST API |
| 适合内容 | 深度分析、教程 | 体验分享、种草 | 技术文章、SEO |

## 一稿多投适配策略

同一话题为不同平台生成差异化内容：

### 步骤 1：确定核心素材

- 用 `trending_topics` 获取话题
- 用 `web_search` 收集数据和案例
- 用 `image` 生成基础配图

### 步骤 2：按平台适配

```
核心素材
├── 公众号版本：800-2000 字深度长文
│   ├── 结构化小标题
│   ├── 数据图表和引用
│   └── 引导关注和转发
├── 小红书版本：200-500 字精华笔记
│   ├── 口语化改写
│   ├── 醒目封面 + 多图
│   └── 话题标签 5-8 个
└── 网站版本：Markdown 格式
    ├── SEO 标题和描述
    ├── 结构化 heading
    └── 内链和外链
```

### 步骤 3：逐平台发布

发布顺序建议：**公众号 → 网站 → 小红书**

原因：

- 公众号支持原创声明，先发可锁定原创权
- 网站发布后可获取链接作为其他平台引用
- 小红书发布时机可配合流量高峰

## 发布流程编排

### 使用 oa-media 工具的标准流程

```
阶段 1：素材收集
├── trending_topics(fetch) → 获取热点
├── web_search() → 补充素材（可并行）
└── image() → 生成配图（可并行）

阶段 2：内容创作（顺序，按平台逐个创建）
├── content_compose(draft, platform="wechat") → 公众号草稿
├── content_compose(draft, platform="xiaohongshu") → 小红书草稿
└── content_compose(draft, platform="website") → 网站草稿

阶段 3：审批
└── 回报主智能体 → 列出所有草稿 ID → 等待审批

阶段 4：发布（审批通过后）
├── media_publish(publish, platform="wechat") → 公众号
├── media_publish(publish, platform="website") → 网站
└── media_publish(publish, platform="xiaohongshu") → 小红书
```

## 质量把控清单

每个平台版本发布前检查：

### 通用检查

- [ ] 内容事实准确，无虚假信息
- [ ] 无占位文本（[TODO]、{{…}}）
- [ ] 内容原创，无抄袭
- [ ] 图片清晰且风格统一

### 公众号专项

- [ ] 标题 ≤64 字符
- [ ] 封面图 900×500px
- [ ] HTML 格式正确
- [ ] 排版符合规范（字号/间距/配色）

### 小红书专项

- [ ] 标题 ≤20 字符
- [ ] 正文 ≤1000 字符
- [ ] 图片 ≤9 张
- [ ] 标签 5-8 个，格式 `#话题#`
- [ ] 口语化表达，有 emoji

### 网站专项

- [ ] Markdown 格式正确
- [ ] SEO 标题和 meta description 已设置
- [ ] 内链和外链可访问
- [ ] heading 层级结构正确

## 发布时机建议

| 平台 | 最佳发布时间 | 原因 |
|------|-------------|------|
| 公众号 | 工作日 8:00-9:00, 12:00-13:00, 20:00-21:00 | 通勤和午休高峰 |
| 小红书 | 19:00-22:00 | 晚间活跃高峰 |
| 网站 | 工作日上午 | SEO 爬虫活跃期 |
