# 小红书内容运营

本技能为 oa-media 子智能体提供小红书平台内容运营的完整知识体系。

## 平台限制

| 项目 | 限制 |
|------|------|
| 标题 | ≤20 字符 |
| 正文 | ≤1000 字符 |
| 图片 | ≤9 张/篇 |
| 标签 | 建议 5-8 个，≤10 个 |
| 操作间隔 | ≥5 秒（RPA 模式） |

## 笔记创作

### 内容原则

1. **真实优先** — 基于真实体验，避免过度营销
2. **视觉驱动** — 图片/视频质量决定第一印象
3. **社区调性** — 亲和、有温度、口语化
4. **提供价值** — 干货分享 > 广告推销

### 内容类型

| 类型 | 适用场景 | 特点 |
|------|---------|------|
| 干货教程 | 方法论、技巧分享 | 高收藏率 |
| 好物推荐 | 产品体验分享 | 高互动率 |
| 经验复盘 | 个人成长、职场 | 高共鸣 |
| 热点解读 | 时事、趋势分析 | 高曝光 |

## 标题公式

小红书标题需在 20 字内精准传达价值。

### 高效公式

1. **数字 + 结果**：「5 步搞定居家收纳」
2. **痛点 + 方案**：「总熬夜？试试这个作息法」
3. **避坑型**：「求求别再踩这 3 个护肤坑了」
4. **惊叹型**：「天哪！这个方法也太好用了」
5. **目标人群 + 场景**：「打工人必看的省钱攻略」

### 标题技巧

- 使用口语化表达
- 适当用 emoji 增加视觉吸引
- 包含核心关键词（利于搜索）
- 制造好奇心或紧迫感

## 封面设计

封面是瀑布流中的第一竞争力。

### 设计要点

- **简洁清晰** — 主体突出，背景干净
- **文字醒目** — 核心信息用大字体标注
- **色彩鲜明** — 使用高饱和度色彩或品牌色
- **比例统一** — 竖版 3:4 或 4:5 最佳

### 封面类型

| 类型 | 适用 | 要点 |
|------|------|------|
| 纯文字封面 | 干货总结 | 关键词 + 修饰词（纯干货/必看） |
| 前后对比 | 美妆/健身 | 效果直观 |
| 产品实拍 | 好物推荐 | 高质量实拍图 |
| 渐变背景 | 通用 | 吸引视觉焦点 |

## 话题标签策略

标签直接影响内容被推荐的范围。

### 标签组合公式

```
2 个核心词标签 + 2-3 个长尾标签 + 1-2 个热门标签
```

### 标签规则

- 格式：`#话题#`
- 与内容强相关（算法惩罚不相关标签）
- 混合大词（>100 万浏览）和小词（精准定位）
- 关注平台"灵感话题"获取流量加权
- 定期更新标签组合，避免重复

## 评论互动

互动数据是算法推荐的核心指标。

### 互动策略

1. **发布后 1 小时内** 回复所有评论（触发二次推荐）
2. **回复要求**：
   - 简洁（≤100 字符）
   - 温暖口语化
   - 针对具体观点回应
   - 适当使用 emoji
3. **引导互动**：在正文末尾设计开放性问题
4. **去重**：不重复回复同一评论
5. **上报**：恶意评论/投诉交由主智能体处理

### 禁止行为

- 不暴露 AI 身份
- 不在评论区做硬广
- 不与用户发生冲突
- 不回复涉及隐私的内容

## 使用 oa-media 工具的流程

### 内容创作流程

```
1. trending_topics → 获取热点
2. web_search → 调研素材
3. 用 LLM 能力按上述公式撰写标题和正文
4. image → 生成封面和配图
5. content_compose(draft, platform="xiaohongshu") → 保存草稿
6. 回报主智能体 → 等待审批
```

### 互动管理流程

```
1. social_interact(list_comments) → 获取新评论
2. 对每条评论用 LLM 生成回复
3. social_interact(reply_comment) → 逐条回复（间隔 ≥5s）
4. social_interact(list_dms) → 获取新私信
5. social_interact(reply_dm) → 回复私信
6. 回报处理摘要
```
