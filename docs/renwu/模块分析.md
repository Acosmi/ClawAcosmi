#### Go 缺失 ❌

**❌ 1. Slack 域名安全验证（L7-45）**

- **TS 函数**：`normalizeHostname()`, `isSlackHostname()`, `assertSlackFileUrl()`
- **作用**：防止 token 泄漏到非 Slack 域名。维护白名单 `["slack.com", "slack-edge.com", "slack-files.com"]`，拒绝非 HTTPS 和非 Slack 主机名
- **Go 现状**：`DownloadSlackMedia()` 直接对 `file.URLPrivateDownload` 发请求，**无任何域名校验**
- **安全影响**：⚠️ **中风险** — 如果恶意构造的消息包含伪造的 `url_private`，Go 端会把 Bot Token 发送到任意 URL
- **Go 补全位置**：[monitor_media.go](file:///Users/fushihua/Desktop/Claude-Acosmi/backend/internal/channels/slack/monitor_media.go) L30 之前

**❌ 2. 跨域重定向 Auth 处理（L60-116）**

- **TS 函数**：`createSlackMediaFetch()`, `fetchWithSlackAuth()`
- **作用**：Slack 文件 URL 会 302 重定向到 CDN（不同域名），Node.js `fetch` 出于安全会在跨域重定向时 strip `Authorization` header。TS 端手动处理：第一次带 Auth + `redirect: "manual"`，跟 302 到 CDN 时不再带 Auth
- **Go 现状**：Go `http.DefaultClient` 默认跟随重定向，但不会在跨域时自动 strip Auth header — 实际上 Go 的行为恰好与 TS 相反，**会把 token 泄漏到 CDN**
- **安全影响**：⚠️ **中风险** — Bot Token 可能泄漏给 Slack CDN 域名
- **Go 补全位置**：[monitor_media.go](file:///Users/fushihua/Desktop/Claude-Acosmi/backend/internal/channels/slack/monitor_media.go) 需自定义 `CheckRedirect` 函数

**❌ 3. 文件大小限制（L130-157）**

- **TS 逻辑**：`resolveSlackMedia()` 接受 `maxBytes` 参数，下载后检查 `buffer.byteLength > maxBytes` 则跳过
- **Go 现状**：无大小限制，无限读取到内存
- **Go 补全位置**：[monitor_media.go](file:///Users/fushihua/Desktop/Claude-Acosmi/backend/internal/channels/slack/monitor_media.go) `DownloadSlackMedia()` 中 `io.Copy` 应改为 `io.LimitReader`

**❌ 4. 文件存储到媒体系统（L144-158）**

- **TS 逻辑**：`saveMediaBuffer()` 将下载内容保存到统一媒体存储系统（`media/store`），返回含 `path`, `contentType`, `placeholder` 的结构
- **Go 现状**：直接 `os.Create` 写到 `destDir`，不经过统一媒体系统
- **Go 补全位置**：待确认 Go 端是否有统一 `media/store` 模块

**❌ 5. 线程起始消息解析 + 缓存（L166-208）**

- **TS 函数**：`resolveSlackThreadStarter()` + `THREAD_STARTER_CACHE`
- **作用**：获取线程首条消息内容（含附件），用于上下文补全。使用 `conversations.replies` API，结果缓存在 `Map<string, SlackThreadStarter>`
