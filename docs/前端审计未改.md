## 五、类型安全问题

### 5.1 `as unknown as` 链式转型（70+ 处）

这是整个前端代码库中**最严重的类型安全问题**。分布如下：

| 文件 | 出现次数 | 说明 |
|------|----------|------|
| `app-gateway.ts` | 20+ | 几乎每个事件处理都需要 `host as unknown as ...` |
| `app-lifecycle.ts` | 12+ | 生命周期钩子函数参数转型 |
| `app.ts` | 8+ | 方法委托调用 |
| `app-render.helpers.ts` | 7 | `state as unknown as OpenAcosmiApp` |
| 其余文件 | 10+ | 散落各处 |

**根因**：`AppViewState` 接口定义不完整，许多属性和方法实际存在于 `OpenAcosmiApp` 类上，但未在 `AppViewState` 中声明。导致所有使用 `state: AppViewState` 参数的函数都需要强制转型才能访问实际属性。

**建议**：

1. 扩展 `AppViewState` 接口，补齐缺失的属性声明
2. 或创建一个完整的 `AppStateWithMethods` 类型
3. 消除所有 `as unknown as` 链

### 5.2 `Record<string, unknown>` 弱类型

| 位置 | 说明 |
|------|------|
| `chat.ts` L490-491 | `message as Record<string, unknown>` — message 对象缺少类型定义 |
| `chat.ts` L543 | 同上 |

### 5.3 过度 Optional Chaining

`usage.ts` 中大量 `usage.messageCounts?.total ?? 0` 等模式，表明数据类型定义不够严格，字段应为 required 但标记为了 optional。

## 六、CSS 架构问题

| 问题 | 详情 | 影响 |
|------|------|------|
| 巨型内联 CSS | `usage.ts` 前 1,916 行是 `usageStylesString` CSS 常量，通过 `<style>` 注入 | 维护困难、无语法高亮、无 autoprefixer |
| inline style 滥用 | `style="padding: 20px; text-align: center"` 等至少出现 **30+ 次** | 无法复用、难以统一修改 |
| 无 CSS 模块 | Lit html 模板未使用 Shadow DOM，class 命名冲突风险 | 样式污染 |
| 重复样式模式 | `.muted + padding + text-align: center` 至少 15 次 | 应提取为 utility class |

**建议**：

- 将 CSS 提取到独立 `.css` 文件或 Lit 的 `static styles`
- 减少 inline style，使用语义化 class
- 建立 utility class 系统（`.empty-state`, `.card-section` 等）

## 七、性能与 Bundle 优化

| 问题 | 详情 | 建议 |
|------|------|------|
| Bundle > 500 kB | 每次 vite build 都警告 chunk 过大 | 使用 `dynamic import()` 对 usage/wizard/config 做 code splitting |
| 无 Code Splitting | 所有 view 打包在一起 | 路由级别 lazy loading |
| 重复计算 | `renderUsage` 每次调用都执行 `computeSessionTotals`, `buildAggregatesFromSessions` 等 | 添加 memo/缓存 |
| SVG inline 重复创建 | `app-render.helpers.ts` 中 refresh/focus icon 是 inline SVG | 提取为静态常量 |
| 无虚拟列表 | `renderSessionsCard` 渲染全部 session（最多 1000） | 添加虚拟滚动 |
