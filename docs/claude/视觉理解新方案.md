# 视觉理解新方案：主-视觉子 Agent 实时交互架构

> 研究结论：B（WebSocket 双向流）+ C（VLA 模型持续输出）+ D（审批集中主端）三方案整合

## 问题背景

### 当前痛点

主 Agent 与视觉子 Agent（Argus）通信存在严重往返问题：

```
主 Agent → tool_use → ArgusBridge.AgentCallTool() → [同步阻塞等待]
    → Argus 截图 → Argus 内部调 LLM → 返回分析
    → 主 Agent 再决策 → 触发审批 → [等待用户确认]
    → 再调 Argus 执行 → [等待]
```

**代码定位**：
- 同步阻塞：`backend/internal/agents/runner/attempt_runner.go` — `ArgusBridgeForAgent.AgentCallTool()`
- 内部 LLM 循环：`backend/internal/browser/pw_ai_loop.go` — `AIBrowseLoop.Run()`
- 审批门：`CoderConfirmationManager`（所有动作均触发，包括纯读操作）
- 视觉模型未激活：`backend/cpp/vision/vision_bridge.cpp` — pass-through 模式

### 国际主流方案对比

| 系统 | 交互模型 | 真实时 | 关键创新 |
|------|---------|--------|---------|
| Anthropic Claude CU | 工具调用循环，阻塞 | ❌ | 子 Agent 上下文隔离 |
| OpenAI Operator | Responses API + SSE | 部分 | 内部多工具链式调用 |
| Google Mariner | 云 VM 轮询 | ❌ | 10 个并行浏览器 VM |
| **Google ADK + Gemini Live** | **WebSocket 双向流** | **✅** | LiveRequestQueue 无轮次边界 |
| OpenHands | 事件流委托 | 部分 | WebSocket 流式事件广播 |
| **ShowUI (CVPR 2025)** | **单模型前向传播** | **接近实时** | VLA 端到端，视觉+推理+动作一次完成 |
| OpenCUA-7B | 反思 CoT 微调 VLM | ❌ | Reflective CoT 减少错误重试 |

**结论**：目前没有系统彻底解决 GUI 自动化的往返问题；最接近的是 ShowUI（端到端 VLA）和 Google ADK（持久 WebSocket 流）。

## 方案 D：审批机制重构（优先级最高，最小改动）

### 核心原则
子智能体无审批权，只负责持续输出带时间戳的观察事件；审批统一由主 Agent 的 `CoderConfirmationManager` 在动作执行前按危险等级判断。

### 当前 vs 新流程对比

```
【当前流程（有问题）】
主 Agent → tool_use → AgentCallTool() → [阻塞等待]
→ Argus 截图 → Argus 内部 LLM → 返回分析
→ 主 Agent 决策 → 触发审批 → [等待用户] → 再调 Argus 执行

【新流程（方案 D）】
ScreenObserver 持续运行（独立于对话流）
    ↓ 每帧写入 ObservationBuffer（带时间戳）
主 Agent 需要视觉状态时：
    ObservationBuffer.LatestKeyframe() → 直接读最新帧（0ms）
    LLM 决策 → 生成 ActionCmd
    CoderConfirmationManager 按危险等级判断：
      ├─ RiskNone（截图/读取）→ 无需审批，直接执行
      ├─ RiskLow（滚动/悬停）→ 可配置自动审批
      ├─ RiskMedium（点击/键入）→ 请求用户确认
      └─ RiskHigh（导航/提交）→ 强制审批
    用户确认 → 主 Agent 发出 ActionCmd → ScreenObserver 执行
```

### 动作危险等级分类

```go
// 设计位置：backend/internal/agents/runner/action_risk.go（新文件）

const (
```

## 方案 C：VLA 模型持续输出（带时间戳）

### VisionObservation 数据结构

```go
// 设计位置：backend/internal/argus/observation.go（新文件）

type VisionObservation struct {
    // 标识
    ID        uint64 `json:"id"`           // 单调递增序列号
    SessionID string `json:"session_id"`
    AgentID   string `json:"agent_id"`

    // 时间（主 Agent 按此查询，关键字段）
    CapturedAt      int64         `json:"captured_at"`        // Unix ms
    CapturedAtMono  int64         `json:"captured_at_mono"`   // 单调时钟 ns，防时钟跳变
    CaptureLatency  time.Duration `json:"capture_latency"`
    VLMInferLatency time.Duration `json:"vlm_infer_latency"`  // VLA 推理耗时

    // 内容
    ScreenshotPNG   []byte `json:"screenshot_png,omitempty"`   // 完整截图（存 buffer）
    ThumbJPEGBase64 string `json:"thumb_b64,omitempty"`        // 64×64 预览（推 WS）
    Width           int    `json:"width"`
    Height          int    `json:"height"`

    // 变化检测
    FrameHash       uint64  `json:"frame_hash"`       // xxhash64(64×64 thumbnail)
    ChangeMagnitude float32 `json:"change_magnitude"` // 0.0~1.0
    IsKeyframe      bool    `json:"is_keyframe"`       // 是否触发 VLM 推理

    // VLM 分析结果（仅关键帧有）
    VLMAction    *VLMActionResult `json:"vlm_action,omitempty"`
    VLMModelID   string           `json:"vlm_model_id,omitempty"`
    Elements     []UIElement      `json:"elements,omitempty"`
    ActiveWindow string           `json:"active_window"`

    // 来源溯源
    TriggerSource  ObservationTrigger `json:"trigger_source"`
    CauseActionID  uint64             `json:"cause_action_id,omitempty"`
}

type VLMActionResult struct {
    Action    string     `json:"action"`    // "CLICK"/"TYPE"/"SCROLL"/"NAVIGATE"/"DONE"
    Value     string     `json:"value"`
    Position  [2]float32 `json:"position"`  // 归一化坐标 [x, y] 0.0~1.0
    XPx       int        `json:"x_px"`      // 转换后像素坐标
    YPx       int        `json:"y_px"`
    Reasoning string     `json:"reasoning"`
    RawOutput string     `json:"raw_output"`
}

type ObservationTrigger string
const (
    TriggerScheduled  ObservationTrigger = "scheduled"    // 定时轮询
    TriggerPostAction ObservationTrigger = "post_action"  // 动作执行后
    TriggerManual     ObservationTrigger = "manual"       // 用户手动触发
)
```

### ScreenObserver 核心循环（生产者）

```go
// 设计位置：backend/internal/argus/screen_observer.go（新文件）

type ScreenObserverConfig struct {
    BaseInterval    time.Duration // 默认 1s（1fps）
    ChangeThreshold float32       // 默认 0.02（2% 像素变化）
    MaxBufferFrames int           // 默认 500 帧
    VLAEnabled      bool
    VLAClient       VLAClient
    Goal            string        // 当前任务目标（传给 VLA）
}

// 核心流程（伪代码）：
// 1. 定时截图
// 2. xxhash64 去重（相同 hash 跳过）
// 3. 计算像素变化幅度
// 4. 关键帧（变化 > threshold）才调 VLA 推理
// 5. 写入 ObservationBuffer（非阻塞）
// 6. 广播到 WS 订阅者（仅元数据+缩略图）
```

### ObservationBuffer 查询接口（主 Agent 使用）

```go
// 设计位置：backend/internal/argus/observation_buffer.go（新文件）

type ObservationBuffer interface {
    Last(n int) []*VisionObservation             // 最新 n 帧
    Since(sinceMs int64) []*VisionObservation    // 指定时间后的所有帧
    LatestKeyframe() *VisionObservation          // 最新关键帧（含 VLM 分析）
    Nearest(targetMs int64) *VisionObservation   // 最接近指定时间戳的帧
    Subscribe(bufSize int) (id uint64, ch <-chan *VisionObservation)
    Unsubscribe(id uint64)
}
```

## 实施路线图

```
Phase 1 — 方案 D（最小改动，立竿见影）
  ├── 定义 ActionRiskLevel，视觉"读取"类操作跳过审批
  ├── 子 Agent 管理菜单（UI）：基础开/关 + 频率设置
  └── 预期收益：消除视觉"读"操作的审批阻断

Phase 2 — 方案 C（独立 ScreenObserver）
  ├── ScreenObserver goroutine + ObservationBuffer ring buffer
  ├── 主 Agent 改为从 Buffer 查询（LatestKeyframe）
  ├── 接入 Anthropic Vision API 作为 VLA（零本地部署，快速验证）
  └── 预期收益：消除主 Agent 等待视觉的往返延迟

Phase 3 — 方案 B（WS 推送）
  ├── WS 推送 observation 事件（元数据 + 缩略图）到前端
  ├── 子 Agent 管理菜单完整功能（实时状态、帧预览、统计）
  └── 预期收益：前端实时展示视觉 Agent 状态

Phase 4 — 方案 C 增强（本地 VLA）
  ├── vLLM 部署 ShowUI-2B 或 OpenCUA-7B（需 GPU）
  ├── 统一 VLAClient 接口切换
  └── 预期收益：低延迟本地视觉推理（<1.5s/帧）
```

## 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 子 Agent 有无审批权 | 无 | 子 Agent 是传感器，不是决策者 |
| 截图是否走 WS 推送 | 否（仅元数据+缩略图） | WS 帧 <10KB，高频不卡顿 |
| Ring Buffer 大小 | 500 帧 | 约 500MB（1080p PNG）；可配置 |
| 去重策略 | xxhash64(64×64 thumbnail) | <1ms，零 VLM 开销 |
| VLA 调用条件 | 仅关键帧（变化>2%） | 控制成本，避免无效推理 |
| 主 Agent 获取视觉方式 | Buffer.LatestKeyframe() | 0ms 延迟，非阻塞 |
| 时间戳精度 | Unix ms (int64) | 避免 ISO 8601 解析开销 |
