# macOS 升级后视觉子智能体恢复指引（MM-008）

更新时间：2026-03-04

## 1. 适用场景

- 系统升级到 macOS Sequoia（15+）后，视觉子智能体（Argus）无法启动或无法截图。
- 表现为“可执行文件存在但权限不通过”“偶发启动后立即失效”。

## 2. 根因分层

- TCC 权限变化：`Screen Recording` / `Accessibility` 在升级后可能失效，Sequoia 存在月度重授权窗口。
- 二进制签名变化：裸二进制重编译后哈希变化，授权无法稳定复用。
- 启动前未做显式预检：问题在首次调用时才暴露，用户侧感知为“随机不可用”。

## 3. 已落地机制

### 3.1 启动前权限预检

- 文件：`backend/internal/argus/tcc_darwin.go`
- 能力：
  - 检测 `Screen Recording` 与 `Accessibility` 状态；
  - 给出可执行恢复文案（`Recovery()`）；
  - 检测 Sequoia 屏幕录制授权剩余天数并提前告警。

### 3.2 签名持久化兜底

- 文件：`backend/internal/argus/codesign_darwin.go`
- 能力：
  - 优先识别 `.app` 内稳定签名路径；
  - 裸二进制在可用条件下自动签名，降低升级/重编译后的授权漂移。

### 3.3 网关可观测接口

- 文件：`backend/internal/gateway/server_methods_argus.go`
- 方法：
  - `argus.permission.check`：返回当前 TCC 状态、恢复提示、剩余天数；
  - `argus.status`：返回 bridge 状态和健康信息。

## 4. 运维处理流程（建议）

1. 调用 `argus.permission.check`，确认缺失权限类型。  
2. 按返回 `recovery` 文案进入系统设置重新授权。  
3. 调用 `argus.status` 或执行一次视觉任务进行回归。  
4. 若仍失败，检查二进制路径与签名日志，再执行重启（`argus.restart`）。  

## 5. 验证证据

- `go test ./internal/argus -run 'TestTCCStatus_|TestCheckTCCPermissions_NoError|TestEnsureCodeSigned|TestCheckBinary'`
- 结果：通过。
